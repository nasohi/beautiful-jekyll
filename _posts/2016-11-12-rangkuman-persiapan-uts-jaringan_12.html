Masalah utama dalam komunikasi antar komputer dari vendor yang berbeda adalah karena mereka mengunakan protocol dan format data yang berbeda-beda. Untuk mengatasi ini, International Organization for Standardization (ISO) membuat suatu arsitektur komunikasi yang dikenal sebagai Open System Interconnection (OSI) model yang mendefinisikan standar untuk menghubungkan komputer-komputer dari vendor-vendor yang berbeda.


Perbedaan antara layer OSI dan TCP/IP ada pada jumlah layernya, layer OSI memiliki 7 layer sedangkan TCP/IP memiliki 4 layer, TCP/IP menggabungkan OSI presentation dan session layer kedalam application layer, TCP/IP menggabungkan osi data link dan physical layer kedalam network access layer dan TCP/IP protocol adalah standart dalam pengembangan internet.

Layer-layer OSI dan fungsinya :
Anak Pak Soleh Tidak Nakal Dan Pintar
7. Application layer -> Berfungsi sebagai antar muka dan mengatur aplikasi yang berhubungan dengan jaringan seperti http,ftp,smtp,nfs.

6. Presentation layer -> Berfungsi untuk mentranslasikan data yang hendak dikirim oleh aplikasi ke dalam format yang bisa ditransmisi melalui jaringan.

5. Session layer -> Berfungsi untuk membuka dan mendefinisikan bagaimana koneksi dibuat (set up), dipelihara (monitor), atau dihancurkan (terminate the connection).

4. Transport layer -> Berfungsi untuk memecah data ke dalam paket-paket untuk dikirim serta memberikan nomor urut ke paket-paket tersebut sehingga dapat disusun kembali ketika paket sudah sampai ditujuan.

3. Network layer -> Berfungsi untuk melihat dan mendifiniskan alamat-alamat IP, membuat header untuk paket-paket, kemudian melakukan routing, dan  mengatur seperti ip,icmp,arp)

2. Data-Link layer -> Berfungsi untuk mengkoreksi kesalahan, flow control untuk MAC (Media Access Control Address) dan LLC (logical layer control) didalam perangkat jaringan

1. Physical layer -> Berfungsi untuk mendefinisikan media transmisi jaringan, sinkorinasasi bit seperti ethernet/NIC melalui media kabel/radio.


TCP/IP Layer
1. Network Access Layer -> bertanggung jawab untuk meletakkan frame-frame(digital data transmission unit in computer networking and telecommunication)  jaringan di atas media jaringan yang digunakan

2. Internet Layer ->  bertanggung jawab untuk melakukan pemetaan (routing) dan enkapsulasi paket-paket data jaringan menjadi paket-paket IP(ip,icmp,arp).

3. Transport ->  berguna untuk membuat komunikasi menggunakan sesi koneksi yang bersifat connection-oriented atau connectionless (TCP dan UDP)

4. Application -> bertanggung jawab untuk menyediakan akses kepada aplikasi terhadap layanan jaringan TCP/IP. (DHCP,FTP,SNMP,DNS,SMTP,HTTP)


Pengertian logical connection dan physical connection.
Logical connection adalah bentuk koneksi yang tidak terlihat, hanya bisa dilihat dengan software packet monitoring misal wireshark, logical connection ada di TCP/IP connection layer by layer.
Physical connection adalah bentuk koneksi yang bisa dilihat wujudnya seperti kabel, perangkat wireless, router, switch, yang bisa dicek terlebih dahulu koneksinya ketika network troubleshooting sebelum masuk ke logical connection.

MAC Address (Media Access Control Address ) adalah alamat hardware yang unik digunakan sebagai identitas perangkat seperti ethernet yang terdapat pada layer 2 OSI data link. MAC Address juga sering disebut sebagai Ethernet address, physical address, atau hardware address. MAC Address menggunakan 48 bit (6 byte).

Port adalah port adalah mekanisme yang mengizinkan sebuah komputer untuk mendukung beberapa sesi koneksi dengan komputer lainnya dan program di dalam jaringan, analoginya seperti sebuah lubang untuk membentuk sebuah koneksi seperti port TCP contoh port 80 untuk membuka koneksi web. Port menggunakan 16 bit (2 byte).


TCP dan UDP

TCP (Transmission Control Protocol)
-Connection Oriented dan reliable
-Berorientasi sambungan (connection-oriented): Sebelum data dapat ditransmisikan antara dua host, dua proses yang berjalan pada lapisan aplikasi harus melakukan negosiasi untuk membuat sesi koneksi terlebih dahulu.
-Dapat diandalkan (reliable): Data yang dikirimkan ke sebuah koneksi TCP akan diurutkan dengan sebuah nomor urut paket dan akan mengharapkan paket positive acknowledgment ( dikirimkan telah diterima dengan sempurna tanpa ada kesalahan) dari penerima.
-Protokol yang digunanakan untuk membuat koneksi seperti http menggunakan TCP port 80, ftp menggunakan port 21.
-Didalam model OSI ataupun model DARPA (TCP/IP) terdapat di layer transport.

UDP (User datagram protokol)
-Connectionless dan unreliable
-Connectionless (tanpa koneksi): Pesan-pesan UDP akan dikirimkan tanpa harus dilakukan proses negosiasi koneksi antara dua host yang hendak bertukar informasi.
-Unreliable (tidak andal): Pesan-pesan UDP akan dikirimkan sebagai datagram tanpa adanya nomor urut atau pesan acknowledgment
-Salah satu protokol TCP/IP
-UDP Port contoh DNS menggunakan port 53, TFTP menggunakan port 69, SMB file sharing 445, 67 BOOTP DHCP Client, 68 BOOTP DHCP Server


ICMP,IGMP,ARP,RARP terdapat di layer network baik di OSI maupun TCP/IP

ARP->   digunakan untuk menemukan physical address dari sebuah komputer ketika internet address sudah diketahui
The Address Resolution Protocol (ARP) is used to associate a logical address with a physical address. On a typical physical network, such as a LAN, each device on a link is identified by a physical or station address, usually imprinted on the network interface card (NIC).
ARP is used to find the physical address of the node when its Internet address is known.

RARP -> kebalikan dari ARP
The Reverse Address Resolution Protocol (RARP) allows a host to discover its Internet address when it knows only its physical address.
It is used when a computer is connected to a network for the first time or when a diskless computer is booted.

Internet Control Message Protocol -> mengecek koneksi jaringan seperti perintah ping
The Internet Control Message Protocol (ICMP) is a mechanism used by hosts and gateways to send notification of datagram problems back to the sender.
ICMP sends query and error reporting messages.

Internet Group Message Protocol
The Internet Group Message Protocol (IGMP) is used to facilitate the simultaneous transmission of a message to a group of recipients.
