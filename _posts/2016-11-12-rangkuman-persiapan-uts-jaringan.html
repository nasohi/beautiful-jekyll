-Karakter dasar data communication
 1. Delivery = System harus mengirim data ke tujuan yang benar.
 2. Accuracy = System harus mengirim data secara akurat.
 3. Timeliness = System harus mengirim data dg waktu yang akurat.

-Komponen-komponen data communication
 1. Message = Informasi atau data yg dikirim/diterima.
 2. Sender = Perangkat yang mengirim data.
 3. Receiver = Perangkat yang menerima data.
 4. Medium = Jalur fisik sebagai perantara untuk mengirimkan data antara sender dan reciever.
 5. Protocol = Sekumpulan aturan-aturan yang mengatur komunikasi data.

-Definisi network = sekumpulan perangkat yang saling terhubung melalui media jaringan untuk berbagi data/informasi

-Distributed processing = Proses yang diterapkan didalam network yang mana membagi tugas ke beberapa komputer.
 Contoh : Website facebook mempunya database yang dibagi ke beberapa komputer server (distributed database) dan bisa diakses dari para pengguna facebook.
 Manfaat :
 1. Security/encapsulation = Dapat membatasi dan mengamankan interaksi antara user dan system.
 2. Distributed database = Membagi system database ke beberapa komputer sehingga bisa meringankan proses.
 3. Faster problem solving = Beberapa komputer bekerja secara bersama dapat menyelesaikan proses lebih cepat daripada satu komputer.
 4. Security through redudancy = Beberapa komputer menjalankan program pada waktu yang sama dapat mengamankan data melalui redundancy.
 5. Collaborative processing = Beberapa komputer dan beberapa user bisa saling berinteraksi pada sebuah program.

-Kriteria Network :
 1. Perfomance = Salah satu kriteria network yang baik memiliki perfomance yang baik, perfomance tergantung pada 4 faktor :
    a. Jumlah pengguna (Semakin banyak user yang mengakses network secara bersamaan dapat memperlambat response time)
    b. Tipe media transmisi (kecepatan dari jenis-jenis media transmisi)
    c. Hardware (semakin bagus cpu,memori,hardisk semakin baik)
    d. Software (sebuah software untuk memaksimalkan network)

 2. Reliability = akurasi dalam pengiriman data diukur oleh :
    a. Frequncy of failure : seberapa sering network itu gagal dalam mengirimkan data.
    b. Rovery time of a network after a failure : waktu untuk memulihkan setelah terjadi network failure.
    c. Catastrophe : membackup network ketika terjadi sesuatu yang tidak terduga seperti bencana alam.

 3. Security = mengamankan data dari unautorized access dan viruses

-Network Application : Instant Messaging, Social Media, Teleconferencing, Email, Academic System, Ebanking, Online Shop, Streaming

-Protokol = sekumpulan aturan-aturan yang mengatur komunikasi data
-Key elements of a protocol terdiri dari 3 :
 1. Syntax = mengacu pada struktur dari format data
 2. Sematics = menerjemahkan bit-bit dan mengontrol dari penerjemahan bit-bit tersebut.
 3. Timing = ketika data dikirim dan seberapa cepat

-Standard = menyediakan sebuah model dan panduan untuk pengembangan network untuk para vendor, manufacturer, government, agencies and other service provider.

-2 Kategori standard komunikasi data
 1. De jure standard = standard yang telah dibuat hukum dan regulasinya
 2. De facto standard = standard yang tidak ada hukum dan regulasinya
    Ada 2 kelas untuk de facto standard
    a. Proprietary (closed standard) = menutup system dari vendor lain, hanya bisa digunakan untuk produk buatan vendor tersebut.
    b. Non proprietary (open standard) = terbuka untuk berbagai system bisa dikembangkan oleh group/komite yang masuk ke public domain.

-Standard Organizations = organisasi-organis yang membuat dan mengembangkan perangkat dan software jaringan.
 1. Komite standard = ISO, ANSI, IEEE, EIA, ITU-T
 2. Forum-forum = ISOC, IETF
 3. Regulatory agencies =

-Teknik switching (pertukaran data/informasi):
  1. Circuit switching = jalur komunikasi antara sender dan receiver tidak berubah selama pertukaran data/informasi
  2. Packet switching = data dibagi menjadi paket-paket kecil ke beberapa nodes(perangkat jaringan) sebelum sampai ke tujuannya


-Line configuration = Cara untuk menghubungkan beberapa device untuk bisa saling berkomunikasi. 2 dua :
  1. Point to point (1 link/dedicated link (1 kabel) antara satu komputer ke satu komputer lainnya)
  2. Multipoint (Shared link(spatially and temporally), beberapa komputer menggunakan link secara bersama-sama)

-Topology = Diagram yang menunjukkan hubungan antara beberapa perangkat jaringan. Jenis-jenis topologi :
   1. Mesh topology (setiap perangkat mempunyai satu link/dedicated link/point to point untuk terhubung ke masih-masih komputer)
   2. Star topology (ada perangkat hub yang menghubungkan beberapa komputer)
   3. Tree (variasi dari topology star)
   4. BUS (ada satu kabel backbone)
   5. RING (membentuk seperti lingkaran)


-Mode transmisi digunakan untuk menentukan arah sinyal antara 2 device yang terhubung, ada 3 :
 1. Simplex = Satu arah saja, hanya salah satu saja yang bisa mengirim, yang satunya lagi hanya menerima. contoh : radio fm, siaran televisi.
 2. Half-Duplex Mode = setiap perangkat bisa mengirim dan menerima tetapi tidak pada saat waktu bersamaan, contoh : walkie-talkie
 3. Full-duplex = 2 perangkat bisa mengirim dan menerima secara bersama-sama, contoh : telepon

Kategori Network berdasarkan size, ownership, distance conver dan physical architecture ada 3 :
 1. LAN (Local Area Network ) biasanya di office, building, atau campus
 2. MAN (Metropolitan Area Network ) contoh cable television network ISP
 3. WAN (Wide Area Network ) contoh Internet (kumpulan dari beberapa network LAN dan MAN)
